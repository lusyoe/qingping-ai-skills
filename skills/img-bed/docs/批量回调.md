## POST 批量上传回调接口（上传第3步-必调）

POST /api/upload/batch-callback

批量上传到OSS为第2步，是在前端或第三方自己去拿着签名上传到OSS。
最后上传完成再回调此接口，用于生成加速链接、数据库更新，并返回图片 id 信息可用于后续删除（必调）
<Icon icon="material-outline-add_alert"/> **注意：** oss_key 不能存在重复，建议最后面的名称使用随机数

> Body 请求参数

```json
{
  "images": [
    {
      "oss_key": "images/lusyoe-test/2024/01/15/abc123.jpg",
      "filename": "image1.jpg",
      "file_size": 1024000,
      "mime_type": "image/jpeg",
      "category": "风景",
      "tags": [
        "自然",
        "山水"
      ],
      "visibility": "public"
    },
    {
      "oss_key": "images/lusyoe-test/2024/01/15/def456.jpg",
      "filename": "image2.jpg",
      "file_size": 2048000,
      "mime_type": "image/jpeg",
      "category": "人物",
      "tags": [
        "肖像"
      ],
      "visibility": "public"
    }
  ]
}
```

### 请求参数

|名称|位置|类型|必选|中文名|说明|
|---|---|---|---|---|---|
|x-api-key|header|string| 否 ||none|
|body|body|object| 是 | Request Data|none|
|» images|body|[object]| 是 ||none|
|»» oss_key|body|string| 是 ||图片在OSS的路径，需以images开头，用户名+日期+随机key|
|»» filename|body|string| 是 ||图片名称，可自定义|
|»» file_size|body|integer| 是 ||图片大小，单位byte|
|»» mime_type|body|string| 是 ||图片类型：image/jpeg、image/png等|
|»» category|body|string| 是 ||图片分类，默认为未分类|
|»» tags|body|[string]| 是 ||图片标签（可设置多个，可选）|
|»» visibility|body|string| 是 ||图片可见性，暂时用不到默认均为公开|
